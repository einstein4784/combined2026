1. Introduction

Welcome to the Combined Insurance Services (St.Lucia) Ltd. Management System. This comprehensive software solution is designed to streamline insurance operations, manage customer relationships, process payments, and generate detailed reports.

1.1 System Overview

The system provides the following core functionalities:

  • Customer Management - Store and manage customer information
  • Policy Management - Create and track insurance policies
  • Payment Processing - Record payments and generate receipts
  • Reporting - Generate various reports including cash statements
  • User Management - Manage system users and access control

1.2 System Requirements

  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • Internet connection
  • Screen resolution: 1024x768 minimum (1920x1080 recommended)

2. Getting Started

2.1 Logging In

1Open your web browser and navigate to the system URL

2Enter your username or email address

3Enter your password

4Click the Sign In button

Note: If you forget your password, contact your system administrator to reset it.

2.2 Navigation

The system uses a sidebar navigation menu on the left side of the screen. The menu includes:

  • Dashboard - Overview and quick statistics
  • Customers - Customer management
  • Policies - Policy management
  • Payments - Payment processing
  • Daily Cash Statement - Daily financial summary
  • User Management - User administration (Admin/Supervisor)
  • Reports - Report generation (Admin/Supervisor)
  • Admin Settings - System configuration (Admin only)
  • Support - Help and contact information

2.3 Logging Out

To log out of the system:

  1. Click on your username in the top-right corner
  2. Select Logout from the dropdown menu

3. Dashboard

The dashboard provides a quick overview of your insurance operations.

3.1 Statistics Cards

The top section displays key metrics:

  • Total Customers - Number of registered customers
  • Active Policies - Number of active insurance policies
  • Outstanding Balance - Total amount owed by all customers
  • Today's Collections - Total payments received today

3.2 Recent Activity

The dashboard shows recent activity including:

  • Recently added customers
  • Recently created policies
  • Recent payments received
Tip: Click on any item in the recent activity lists to view more details.

4. Customer Management

4.1 Viewing Customers

Navigate to Customers from the sidebar to view all customers in a searchable table.

4.2 Adding a New Customer

1Click the Add Customer button

2Fill in the required information:

  • First Name, Middle Name (optional), Last Name
  • Address
  • Contact Number
  • Email Address
  • Sex (Male/Female/Other)
  • ID Number

3Click Save Customer

4.3 ID Scanning Feature

The system includes an optional ID scanning feature that uses OCR (Optical Character Recognition) to automatically extract information from ID cards:

  1. Click the Scan ID button next to the ID Number field
  2. Select an image file of the ID card
  3. Wait for the system to process the image
  4. Review and verify the auto-filled information

4.4 Editing a Customer

  1. Find the customer in the table
  2. Click the Edit button (pencil icon)
  3. Modify the necessary information
  4. Click Save Customer

4.5 Deleting a Customer

  1. Find the customer in the table
  2. Click the Delete button (trash icon)
  3. Confirm the deletion when prompted
Warning: Deleting a customer will not delete associated policies or payments. Exercise caution when deleting records.

5. Policy Management

5.1 Understanding Policies

A policy represents an insurance agreement with a customer. Each policy includes:

  • Policy Number - Unique identifier (auto-generated if not provided)
  • Customer - The policyholder
  • Coverage Type - Third Party or Fully Comprehensive
  • Coverage Period - Start and end dates
  • Premium - Total amount due
  • Outstanding Balance - Amount still owed

5.2 Creating a New Policy

1Click the Create Policy button

2Select the customer from the dropdown (searchable)

3Enter or auto-generate the policy number

4Select coverage type:

  • Third Party - Basic liability coverage
  • Fully Comprehensive - Full coverage including own damage

5Set coverage start and end dates

6Enter the total premium amount

7Click Save Policy

5.3 Policy Status

Status Description
Active Policy is currently in effect
Expired Coverage period has ended
Cancelled Policy has been cancelled

5.4 Editing a Policy

  1. Find the policy in the table
  2. Click the Edit button
  3. Modify the necessary information
  4. Click Save Policy

6. Payment Processing

6.1 Recording a Payment

1Navigate to Payments

2Click New Payment

3Select the policy from the dropdown

4Review the outstanding balance displayed

5Enter the payment amount

6Select payment method(s):

  • Cash
  • Credit Card
  • Debit Card
  • Bank Transfer
  • Cheque

7Click Record Payment

Note: Multiple payment methods can be selected for a single transaction.

6.2 Viewing Receipts

After recording a payment, a receipt is automatically generated. To view a receipt:

  1. Find the payment in the payments table
  2. Click the View Receipt button
  3. The receipt will open in a new page
  4. Use the print button to print or save as PDF

6.3 Outstanding Balance Rule

Business Rule: A payment cannot be received for a customer that has an outstanding balance on another policy unless the arrears are cleared or an override is granted by an authorized user.

6.4 Payment History

The payments table shows all payment records including:

  • Receipt Number
  • Policy Number
  • Customer Name
  • Amount
  • Payment Method
  • Date and Time

7. Reports

The Reports section allows you to generate various reports for analysis and record-keeping.

7.1 Available Reports

Report Type Description
Cash Statement Summary of all payments received for a date range
User Activity Detailed log of user actions in the system
Policy Report List of policies with coverage and payment details
Payment Report Detailed payment transactions
Customer Report Customer list with policy and balance information

7.2 Generating Reports

1Navigate to Reports

2Select the report type

3Choose a date range or use quick buttons:

  • Today - Current day only
  • This Week - Current week
  • This Month - Current month
  • All Time - All available data

4Click Generate Report

7.3 Exporting Reports

Once a report is generated, you can:

  • Print - Send directly to printer
  • Export PDF - Save as PDF file
  • Back - Return to reports page

7.4 Daily Cash Statement

The Daily Cash Statement is a quick access report available from the main menu. It shows:

  • All payments received on the selected date
  • Breakdown by payment method
  • Total collections

8. User Management

User Management is available to Administrators and Supervisors.

8.1 Creating a New User

1Navigate to User Management

2Click Add User

3Enter user details:

  • Username (unique)
  • Full Name
  • Email Address
  • Password
  • Role

4Click Save User

8.2 Editing a User

  1. Find the user in the table
  2. Click the Edit button
  3. Modify the necessary information
  4. Click Save User
Note: Leave the password field empty when editing to keep the existing password.

8.3 Deleting a User

Important: The default admin account cannot be deleted. The system must always have at least one administrator.

9. Admin Settings

Admin Settings is accessible only to users with the Administrator role.

9.1 Role Permissions

Administrators can customize which functions each role can access:

  1. Select a role tab (Underwriter, Cashier, Supervisor)
  2. Check or uncheck the desired permissions
  3. Click Save Permissions
Note: Admin role always has full access and cannot be modified.

9.2 Financial Periods

Financial periods help organize transactions by accounting periods:

  • Create Period - Start a new financial period
  • Close Period - Close the current period (prevents further changes)

9.3 Data Management

Administrators can export and import data:

  • Export Data - Download system data as CSV for backup
  • Import Data - Upload CSV data to restore or migrate

9.4 System Functions

Danger Zone: The following functions should be used with extreme caution:
  • Reset System - Permanently deletes ALL data
  • Override Arrears - Bypass the outstanding balance rule
